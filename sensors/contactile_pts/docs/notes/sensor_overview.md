# Contactile PTS (PapillArray) 产品概述

> 基于原厂手册 PTSDK_2.0_MAN、PTS_2.1_SPEC_JAN25、PTSCTR_2.0_SPEC 的中文总结。

---

## 1. 系统组成

| 组件 | 数量 | 说明 |
|------|------|------|
| PapillArray 触觉传感器 (v2.1) | 最多 2 个 | 3×3 柱阵列，每传感器 9 个 sensing pillars |
| 控制器 (Controller v2.0) | 1 个 | 协调最多 2 个传感器的数据采集与供电 |
| 传感器线缆 | 2 条 | Molex PicoBlade 10pin 母对母，600mm |
| USB 线缆 | 1 条 | micro-USB → USB，1m |

控制器通过**单根 USB 线**同时完成：供电 + 数据传输。无需额外电源。

## 2. 传感器规格 (PTS v2.1)

### 2.1 物理尺寸

| 参数 | 数值 |
|------|------|
| 外壳尺寸 (W×L×H) | 24.0 × 30.6 × 8.6 mm |
| 材质 | 阳极氧化铝 |
| 安装方式 | 底部 4×M3 螺纹孔 |
| 柱高 | 4.2 mm（中心柱最高，角柱最短） |
| 柱直径 | 6 mm |
| 柱间距 | 7 mm（中心距，正方形网格） |
| 柱材质 | 硅胶，邵氏硬度 A40 |

### 2.2 感知性能

**单柱位移量程**

| 方向 | 量程 | 分辨率 |
|------|------|--------|
| X, Y | ±1 mm | < ±0.01 mm |
| Z | +2.5 mm | < ±0.01 mm |

**单柱力量程**

| 方向 | 量程 | 分辨率 |
|------|------|--------|
| Fx, Fy | ±4 N | < ±0.05 N |
| Fz | 15 N | < ±0.05 N |

**电气与采样**

| 参数 | 数值 |
|------|------|
| 单柱采样率 | 1000 Hz @ 16-bit |
| 抗混叠滤波器截止频率 | 338.8 Hz（-3dB @ 235 Hz） |
| 供电 | USB 5V / 500mA 足够 |

### 2.3 测量能力总结

每个柱可输出：
- **3D 位移** (dx, dy, dz)
- **3D 力** (fx, fy, fz)
- **振动** (隐含在高速采样中)
- **接触状态** (in_contact)
- **滑动状态** (slip_state)

每个传感器可输出：
- **全局 3D 力** (gfx, gfy, gfz)
- **全局 3D 力矩** (gtx, gty, gtz) — 参考点为**中心柱 P4 的当前顶端位置**
- **摩擦估计** (friction_est)
- **目标夹持力** (target_grip_force)

## 3. 控制器规格 (Controller v2.0)

| 参数 | 数值 |
|------|------|
| 尺寸 (W×L×H) | 46 × 36 × 16 mm |
| 材质 | 阳极氧化铝 |
| 安装方式 | 底部 4×M3 螺纹孔 |
| 传感器端口 | SEN0、SEN1 |
| 数据接口 | micro-USB（虚拟串口） |
| 最大电流 | 约 200 mA（2 个传感器时） |

### LED 状态指示灯

**HUB LED**（控制器状态）

| 状态 | 含义 |
|------|------|
| 灭 | 未通电 |
| 白色闪烁 | 等待串口连接 |
| 白色常亮 | 正常工作，正在采样 |
| 红色常亮 | 错误（初始化失败或电压过低） |

**SEN0 / SEN1 LED**（传感器状态）

| 状态 | 含义 |
|------|------|
| 灭 | 未连接传感器（若 HUB 亮） |
| 白色常亮 | 正在采样 |
| 红色常亮 | 传感器错误（数据无效或断开） |

## 4. 重要安全与使用注意事项

### ⚠️ 绝对禁止
- **不要拆解传感器** — 会损坏传感器并**使校准失效**
- **不要钻孔/攻丝/加工外壳** — 会使保修失效
- **不要在磨蚀表面或尖锐边缘上使用** — 会损坏硅胶柱面

### ⚠️ 环境影响
- **强光**：可能导致读数出现小偏移
- **强电磁场**（如 MRI）：可能干扰传感器工作
- **温度变化**：会导致读数漂移。虽有部分温度补偿，但**每次使用前必须进行 Bias（零点校准）**

### ⚠️ Bias（零点校准）要点
- **必须确保传感器完全无负载**时进行 Bias
- Bias 前保持无负载至少 **1 秒**，以消除迟滞效应
- Bias 操作耗时最长 **2 秒**，期间必须保持无负载
- **建议每次确认传感器无负载时都执行 Bias**（尤其前 20 分钟升温阶段）

### ⚠️ 线缆注意事项
- 传感器线缆**是校准系统的一部分**，不可随意更换长度或类型
- 避免线缆过度弯曲、拉扯、被踩踏或被扎带过紧挤压
- 不要频繁插拔传感器线缆

## 5. 通信接口

- **物理层**：USB 虚拟串口（CDC ACM）
- **Linux 设备**：通常出现在 `/dev/ttyACM0`
- **波特率**：
  - C++ / ROS2 默认：**9600**
  - Python 默认：**115200**
- **数据格式**：二进制数据包，8-bit 字节
- **多字节**：小端模式 (Little-Endian, LSB first)

> 详细通信协议参见 [communication_protocol.md](communication_protocol.md)

### 关于 `baud_rate`

PTS 控制器在主机侧同样表现为 USB CDC ACM 虚拟串口，因此这里要区分：

- 主机程序仍然通过串口兼容 API 打开 `/dev/ttyACM*`，并传入 `baud_rate`、`parity`、`byte_size`；
- 但这并不自动等于“主机和控制器之间真的在跑某个物理 UART 波特率”。

当前原厂资料中同时存在两套写法：

- C++ / ROS / ROS2 文档常写 `9600`
- Python 示例和 vendor 示例常写 `115200`

在没有更明确固件说明前，更稳妥的理解是：

- 这些值更像 USB CDC 串口兼容层中的 line-coding 参数；
- 对主机到控制器的 USB 链路来说，不应直接把 `9600` 或 `115200` 表述成真实物理传输速率；
- 本仓库默认沿用各自原厂示例：C++ / ROS2 用 `9600`，Python 示例保留 `115200`。
