# 通信协议笔记

> 基于原厂手册 PTSCOM_2.0_SPEC_SEP22 的中文总结。

---

## 1. 数据包结构

Controller 通过 USB 虚拟串口以**二进制格式**流式传输数据。

每个采样数据为一个**数据包 (data packet)**，结构如下：

```
[ 起始字节 ×4 ] [ 数据载荷 ] [ 校验和 ×2 ] [ 结束字节 ×4 ]
```

| 字段 | 字节数 | 值 | 说明 |
|------|--------|-----|------|
| 起始字节 | 4 | 0x55, 0x66, 0x77, 0x88 | 固定起始标记 |
| 数据载荷 | N | — | 传感器数据（见下文） |
| 校验和 | 2 | — | 数据载荷的截断和 (truncated sum) |
| 结束字节 | 4 | 0xAA, 0xBB, 0xCC, 0xDD | 固定结束标记 |

### 校验和计算

1. 计算数据载荷中每个字节的和（**不包含**起始字节、结束字节和校验和本身）
2. 将结果截断为 **2 字节无符号整数**
3. 与收到的校验和字节比较，验证数据完整性

### 读取流程

1. 逐字节读取，直到连续匹配 4 个起始字节
2. 继续逐字节读取并存入缓冲区，直到连续匹配 4 个结束字节
3. 计算并验证校验和
4. 解析数据载荷

---

## 2. 数据载荷解析要点

- **索引从 0 开始**：起始字节后的第一个字节为索引 0
- **多字节数据采用小端模式** (LSB first)
- 原厂 SDK（C++/Python）已经封装了解析逻辑，一般不需要手动处理此协议

---

## 3. ASCII 控制命令

Controller 接受以下 **ASCII 字符命令**（通过串口发送）：

| 命令 | 说明 | 重要提示 |
|------|------|----------|
| `z\n` | **Bias 传感器** | 耗时最长 2 秒，期间必须保持传感器**无负载** |
| `S\n` | **启动滑动检测** | 需在部分柱已接触、**切向加载前**调用 |
| `s\n` | **停止滑动检测** | 停止并重置滑动检测算法 |
| `f100\n` | 设置采样率 100 Hz | — |
| `f250\n` | 设置采样率 250 Hz | — |
| `f500\n` | 设置采样率 500 Hz | — |
| `f1000\n` | 设置采样率 1000 Hz | — |

### Bias 命令详解

- 目的：消除柱体读数的零点偏移
- 时机：连接串口并开始数据流后，传感器**无负载**时
- 持续时间：最长 2 秒
- 频率：建议**每次确认传感器无负载时**都执行 Bias

### 滑动检测命令详解

- `S\n` 启动条件：部分柱已接触表面，且**尚未施加切向载荷**
- 适用范围：仅检测**切向滑动**，不适用于扭转滑动 (torsional slip)
- `s\n` 停止后会重置算法状态

---

## 4. 默认上电状态

- **采样率**：1000 Hz（上电默认值）
- 可通过 `fXXX\n` 命令或 SDK 的 `setSamplingRate()` 修改
